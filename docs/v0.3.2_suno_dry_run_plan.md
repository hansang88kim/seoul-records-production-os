# v0.3.2 Suno One-Song Dry-Run Plan

## Selected Wrapper

**gcui-art/suno-api** (Node.js / TypeScript)

Rationale (from Provider Research Matrix):
- Uses user's own Suno account credits via SUNO_COOKIE
- Runs locally (`npm run dev` → localhost:3000)
- Custom mode: title, lyrics, tags, negative_tags, instrumental, model
- Polling via `/api/get`
- Returns 2 candidates per generation
- LGPL-3.0 license
- Active maintenance (2026)

### Install

```bash
git clone https://github.com/gcui-art/suno-api.git
cd suno-api
npm install
```

### Local Run

```bash
# Set cookie in .env (inside suno-api directory, NOT Seoul Records .env)
echo "SUNO_COOKIE=your_cookie_here" > .env

# Start server
npm run dev
# → http://localhost:3000
```

### Required Environment Variables (Seoul Records .env)

```env
COMPOSER_PROVIDER=local_unofficial_suno
SUNO_LOCAL_API_BASE_URL=http://localhost:3000
SUNO_COOKIE=                          # User's Suno session cookie
SUNO_DOWNLOAD_FORMAT=wav
ALLOW_MP3_FOR_DISTRIBUTION=false
ALLOW_THIRD_PARTY_SUNO=false
```

### API Endpoints Used

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/custom_generate` | POST | Submit song with title/lyrics/tags |
| `/api/get?ids=id1,id2` | GET | Poll generation status + get metadata |
| `/api/get_limit` | GET | Check remaining credits |
| Audio CDN URL | GET | Download MP3/WAV from Suno CDN |

### Auth / Session

- Cookie extracted from browser DevTools (Network tab → `__clerk_api_version` request → Cookie header)
- Cookie passed via `SUNO_COOKIE` env var
- gcui-art/suno-api forwards cookie to Suno's internal API
- Session expires periodically → user must refresh cookie

### Risk Assessment

| Risk | Mitigation |
|---|---|
| Session cookie expires | Detect 401/403 → return `auth_required`, fallback to ManualImport |
| CAPTCHA triggered | Detect in response body → return `captcha_required`, stop |
| Suno API changes | Provider adapter is isolated — only `local_unofficial_suno.py` changes |
| WAV URL unavailable | Attempt `wav_url` field → if missing, download MP3 as preview only |
| Rate limiting | Enforce 5-min interval between generations, detect 429 |
| Cookie logged accidentally | `safe_error()` redacts all credential-like fields |

### Dry-Run Procedure

1. User starts gcui-art/suno-api locally (`npm run dev`)
2. User sets `SUNO_COOKIE` in Seoul Records `.env`
3. Run: `python workflows/suno_one_song_dry_run.py`
4. Script submits 1 song → polls → downloads candidates → saves report
5. No Streamlit UI required for dry-run

### Mock Mode

For CI and testing, the dry-run script supports `--mock` flag:
```bash
python workflows/suno_one_song_dry_run.py --mock
```
This uses simulated HTTP responses — no real Suno server needed.
