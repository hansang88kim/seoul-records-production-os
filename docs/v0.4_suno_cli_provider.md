# SunoCliProvider — paperfoot/suno-cli Integration Guide

## Overview

SunoCliProvider is a subprocess adapter that wraps [paperfoot/suno-cli](https://github.com/paperfoot/suno-cli), a Rust binary that talks directly to Suno's API. It is the primary automated provider for Seoul Records Production OS v0.4+.

Key advantages over gcui-art/suno-api:
- No CAPTCHA solving service needed (no 2Captcha)
- No Playwright / browser automation for generation
- Auto-auth from your browser's existing login session
- Full Suno v5.5 parameter support (vocal, weirdness, style-influence, persona, exclude)
- JSON output for all commands (`--json` flag)
- Single binary — no Node.js / npm needed

Limitation:
- Downloads MP3 only. WAV requires Suno Pro/Premier subscription and must be downloaded from suno.com manually, then imported via ManualImportProvider.

## Windows Installation

### Option A: Download pre-built binary

1. Go to [github.com/paperfoot/suno-cli/releases](https://github.com/paperfoot/suno-cli/releases)
2. Download the Windows binary (`suno-x86_64-pc-windows-msvc.zip`)
3. Extract `suno.exe` to a folder on your PATH (e.g. `C:\Users\<you>\bin\`)
4. Verify: `suno --version`

### Option B: Install via Cargo (Rust)

```powershell
cargo install suno-cli
```

### Option C: Homebrew (macOS/Linux)

```bash
brew tap 199-biotechnologies/tap
brew install suno-cli
```

## Authentication

```bash
# Auto-extract session from Chrome/Arc/Brave/Firefox/Edge
# (you must be logged into suno.com in your browser)
suno auth --login

# Verify auth status
suno auth

# Check credits
suno credits
suno credits --json
```

Auth is stored in a local file with 0600 permissions. Sessions last approximately 7 days before needing refresh.

To refresh:
```bash
suno auth --refresh
```

## Seoul Records OS Configuration

In your Seoul Records `.env` file:

```env
COMPOSER_PROVIDER=suno_cli
SUNO_CLI_BIN=suno
SUNO_DOWNLOAD_FORMAT=mp3
ALLOW_MP3_FOR_DISTRIBUTION=false
```

`SUNO_CLI_BIN` defaults to `suno` (looks on PATH). Set to a full path if the binary is elsewhere:
```env
SUNO_CLI_BIN=C:\Users\hansang88kim\bin\suno.exe
```

## One-Song Dry-Run

### Quick test (CLI directly)

```powershell
suno generate `
  --title "밤이 지나면" `
  --tags "city pop, female vocal, DX7, chorus guitar, A minor" `
  --exclude "sax lead, trot, EDM, drum fill-in" `
  --lyrics "가사를 여기에" `
  --vocal female `
  --weirdness 35 `
  --style-influence 70 `
  --wait --download ./test_songs/ `
  --json
```

### Via Seoul Records dry-run script

```powershell
# Set provider to suno_cli in .env
# COMPOSER_PROVIDER=suno_cli

# Run dry-run (uses Seoul Records test prompt)
python workflows/suno_one_song_dry_run.py

# Or mock mode (no real Suno, for testing)
python workflows/suno_one_song_dry_run.py --mock
```

Output:
```
outputs/dry_runs/YYYY-MM-DD_suno_one_song/
├─ dry_run_report.json
├─ dry_run_log.txt
├─ candidates/
│  ├─ candidate_A_preview.mp3
│  └─ candidate_B_preview.mp3
└─ manual_import_required.txt   (WAV not available via CLI)
```

## MP3 Download and WAV Distribution

SunoCliProvider downloads MP3 files. For the Seoul Records distribution pipeline:

| File | Source | Distribution |
|------|--------|:---:|
| candidate_A.mp3 | suno-cli `--download` | ❌ YouTube preview only |
| candidate_A.wav | suno.com manual download | ✅ Distribution master eligible |

**Workflow:**
1. `suno generate --wait` → generates song, returns clip IDs
2. `suno download <id>` → MP3 saved locally (preview/YouTube only)
3. Go to suno.com → find the song → download WAV (Pro/Premier required)
4. Import WAV via Seoul Records Song Generation tab → Manual WAV Import
5. Audio QC → distribution_master.wav → video → packages

## Error Handling

| suno-cli error | Seoul Records status | Action |
|----------------|---------------------|--------|
| `auth_expired` | `auth_required` | Run `suno auth --login` |
| `auth_missing` | `auth_required` | Run `suno auth --login` |
| Binary not found | `provider_unavailable` | Install suno-cli |
| Timeout | `polling_timeout` | Retry or check network |
| `rate_limit` | `rate_limited` | Wait before retrying |

## Security Policy

- No CAPTCHA bypass. No 2Captcha.
- Credentials are never logged or committed to GitHub.
- `safe_error()` automatically redacts cookie, token, jwt, session, auth, credential, key, secret, password fields.
- Auth files stored by suno-cli are local-only (chmod 0600).
- `.gitignore` excludes: `.env`, `cookies/`, `tokens/`, `sessions/`, `*.cookie`, `*.token`
