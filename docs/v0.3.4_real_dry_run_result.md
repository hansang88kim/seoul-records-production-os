# v0.3.4 Real Suno Dry-Run Result

Date: 2026-06-28
Wrapper: gcui-art/suno-api (Node.js, localhost:3000)
OS: Windows 11
Seoul Records OS: v0.3.3

## Test Results

### get_limit — PASS ✅

```
GET http://localhost:3000/api/get_limit
Response: {"credits_left":10000,"period":"year","monthly_limit":10000,"monthly_usage":0}
```

Session authentication and credit check work correctly with SUNO_COOKIE.

### custom_generate — FAIL ❌

```
POST http://localhost:3000/api/custom_generate → 500

Server log:
1. Getting the session ID         ← OK
2. KeepAlive...                   ← OK
3. CAPTCHA required               ← Suno triggers hCaptcha
4. Launching browser...           ← Playwright starts
5. Storage.setCookies: Invalid cookie fields  ← CRASH
```

**Root cause (two layers):**

1. **Primary:** Suno requires hCaptcha on every generation request (especially on Windows).
   gcui-art/suno-api needs a paid 2Captcha API key (`TWOCAPTCHA_KEY`) to solve these automatically.

2. **Secondary:** Even if 2Captcha was configured, the Playwright browser cookie injection
   fails with `Invalid cookie fields` — a cookie parsing bug in the wrapper's
   `SunoApi.launchBrowser()` method.

## Decision

| Item | Decision |
|------|----------|
| gcui-art/suno-api | **Not production-safe.** Experimental blocked. |
| get_limit endpoint | Usable for credit checking only |
| custom_generate | Blocked by CAPTCHA + cookie injection bug |
| 2Captcha integration | **Not pursued.** Seoul Records policy: no CAPTCHA bypass. |
| ManualImportProvider | **Remains production-safe default.** |
| Next candidate | paperfoot/suno-cli (v0.4 — Rust CLI, no CAPTCHA service needed) |

## Fallback Strategy

```
LocalUnofficialSunoProvider (gcui-art/suno-api)
  ├─ get_limit → works (credit check only)
  └─ custom_generate → CAPTCHA blocked
      └─ Fallback: ManualImportProvider
          └─ User downloads WAV from suno.com → imports via UI

Future (v0.4):
  SunoCliProvider (paperfoot/suno-cli)
  ├─ Auto-extracts credentials from browser
  ├─ No CAPTCHA solving service needed
  ├─ Full v5.5 parameter support
  └─ Fallback: ManualImportProvider
```

## Recommendation for Users

Until SunoCliProvider is implemented in v0.4:

1. Generate songs directly at suno.com
2. Download WAV files
3. Import via Seoul Records OS Song Generation tab → Manual WAV Import
4. The rest of the pipeline (QC → distribution master → video → packages) is fully automated
